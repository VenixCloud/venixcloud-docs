The Websites API enables you to deploy and manage web-facing services on VenixCloud, from simple static HTML pages to full-stack Node.js, PHP, and Python web applications. Every website you create receives a free `venixcloud.com` subdomain with TLS automatically configured. You can also attach a custom domain through the VenixCloud dashboard after deployment. All requests require a valid API key in the `Authorization` header.

---

## List All Websites

Retrieve every website service associated with your account. The response includes the live URL and any custom domain attached to each site, giving you a quick overview of all your web properties.

<CodeGroup>

```bash cURL
curl -X GET https://api.venixcloud.com/v1/websites \
  -H "Authorization: Bearer YOUR_API_KEY"
```

</CodeGroup>

### Response

**Status: `200 OK`**

```json
{
  "data": [
    {
      "id": "web_ghi789",
      "name": "landing-page",
      "type": "static",
      "url": "https://landing-page.venixcloud.com",
      "status": "running",
      "custom_domain": "www.mycompany.com"
    },
    {
      "id": "web_abc123",
      "name": "api-dashboard",
      "type": "nodejs",
      "url": "https://api-dashboard.venixcloud.com",
      "status": "running",
      "custom_domain": null
    }
  ],
  "total": 2
}
```

<ResponseField name="data" type="array">
  An array of website service objects.

  <Expandable title="Website object fields">
    <ResponseField name="id" type="string">
      The unique identifier for the website, prefixed with `web_`.
    </ResponseField>
    <ResponseField name="name" type="string">
      The human-readable name you assigned to the website service.
    </ResponseField>
    <ResponseField name="type" type="string">
      The website category. One of `static`, `nodejs`, `php`, or `python`.
    </ResponseField>
    <ResponseField name="url" type="string">
      The VenixCloud-provided subdomain URL where the site is publicly accessible.
    </ResponseField>
    <ResponseField name="status" type="string">
      The current operational state. One of `running`, `stopped`, `building`, `provisioning`, or `failed`.
    </ResponseField>
    <ResponseField name="custom_domain" type="string | null">
      A custom domain attached to this website, or `null` if none has been configured.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="total" type="integer">
  The total number of website services returned.
</ResponseField>

## Create a Website

Deploy a new website service by specifying the site type, build configuration, and target region. For static sites, provide a build command and output directory. For dynamic applications, provide a start command as well. VenixCloud builds your site, serves it through its global edge network, and returns the assigned subdomain URL.

<Tip>
  For static sites built with frameworks like Vite, Next.js (static export), or Astro, set `build_command` to `npm run build` and `output_dir` to your framework's default output folder (e.g., `dist` or `out`).
</Tip>

<CodeGroup>

```bash cURL — Static Site
curl -X POST https://api.venixcloud.com/v1/websites \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "landing-page",
    "type": "static",
    "build_command": "npm run build",
    "output_dir": "dist",
    "region": "sa-east-1"
  }'
```

```bash cURL — Node.js App
curl -X POST https://api.venixcloud.com/v1/websites \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "api-dashboard",
    "type": "nodejs",
    "build_command": "npm run build",
    "output_dir": "dist",
    "start_command": "node server.js",
    "region": "us-east-1"
  }'
```

</CodeGroup>

### Request Body

<ParamField body="name" type="string" required>
  A unique name for the website service. This name is also used to generate the subdomain URL, e.g. `my-site` becomes `https://my-site.venixcloud.com`.
</ParamField>

<ParamField body="type" type="string" required>
  The website category to deploy. Accepted values are `static`, `nodejs`, `php`, and `python`.
</ParamField>

<ParamField body="build_command" type="string">
  The shell command to run during the build phase before your site is served. Examples: `npm run build`, `pip install -r requirements.txt`. Leave blank if your project requires no build step.
</ParamField>

<ParamField body="output_dir" type="string">
  The directory containing the built assets that VenixCloud should serve. Examples: `dist`, `build`, `public`. Required when `type` is `static`.
</ParamField>

<ParamField body="start_command" type="string">
  The command used to start your web server process. Examples: `node server.js`, `gunicorn app:app`, `php -S 0.0.0.0:8080`. Required for dynamic site types (`nodejs`, `php`, `python`).
</ParamField>

<ParamField body="region" type="string" required>
  The primary region for the website service. Accepted values are `sa-east-1` (São Paulo) and `us-east-1` (North Virginia).
</ParamField>

### Response

**Status: `201 Created`**

```json
{
  "id": "web_ghi789",
  "name": "landing-page",
  "type": "static",
  "url": "https://landing-page.venixcloud.com",
  "status": "building",
  "region": "sa-east-1",
  "build_command": "npm run build",
  "output_dir": "dist",
  "created_at": "2024-03-18T12:00:00Z"
}
```

<ResponseField name="id" type="string">
  The unique identifier for the newly created website, prefixed with `web_`.
</ResponseField>
<ResponseField name="name" type="string">
  The name you provided in the request.
</ResponseField>
<ResponseField name="type" type="string">
  The website type you specified.
</ResponseField>
<ResponseField name="url" type="string">
  The VenixCloud subdomain URL assigned to this website. The site will be accessible at this URL once the build completes and the status transitions to `running`.
</ResponseField>
<ResponseField name="status" type="string">
  The initial status after creation. Typically `building` for sites with a build step, or `provisioning` for others.
</ResponseField>
<ResponseField name="region" type="string">
  The region the website is deployed to.
</ResponseField>
<ResponseField name="build_command" type="string">
  The build command applied during deployment.
</ResponseField>
<ResponseField name="output_dir" type="string">
  The directory VenixCloud serves static assets from.
</ResponseField>
<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the website service was created.
</ResponseField>

<Note>
  It may take a few minutes for the DNS and TLS certificate to fully propagate after the first deployment. The `url` field is reserved immediately but the site may briefly return a 502 until provisioning is complete.
</Note>
