The Services API is the central entry point for managing everything you host on VenixCloud. Whether you are running a database, a bot, a website, or a custom application, every hosted resource is represented as a service. You can list all your services, create new ones, inspect an individual service, or permanently delete one — all through the endpoints documented below. All requests must include a valid API key in the `Authorization` header.

---

## List All Services

Retrieve a paginated list of every service associated with your account. The response includes each service's current status and region so you can quickly audit your infrastructure at a glance.

<CodeGroup>

```bash cURL
curl -X GET https://api.venixcloud.com/v1/services \
  -H "Authorization: Bearer YOUR_API_KEY"
```

</CodeGroup>

### Response

```json
{
  "data": [
    {
      "id": "svc_abc123",
      "name": "my-postgres-db",
      "type": "database",
      "status": "running",
      "region": "sa-east-1",
      "created_at": "2024-01-15T10:30:00Z"
    }
  ],
  "total": 1
}
```

<ResponseField name="data" type="array">
  An array of service objects belonging to your account.

  <Expandable title="Service object fields">
    <ResponseField name="id" type="string">
      The unique identifier for the service, prefixed with `svc_`.
    </ResponseField>
    <ResponseField name="name" type="string">
      The human-readable name you assigned to the service.
    </ResponseField>
    <ResponseField name="type" type="string">
      The service category. One of `database`, `bot`, `website`, or `custom`.
    </ResponseField>
    <ResponseField name="status" type="string">
      The current operational state. One of `running`, `stopped`, `building`, `failed`, or `provisioning`.
    </ResponseField>
    <ResponseField name="region" type="string">
      The region where the service is deployed, e.g. `sa-east-1` or `us-east-1`.
    </ResponseField>
    <ResponseField name="created_at" type="string">
      ISO 8601 timestamp indicating when the service was created.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="total" type="integer">
  The total number of services returned in this response.
</ResponseField>

## Create a Service

Provision a new service on VenixCloud by specifying a name, type, and target region. You can optionally provide a runtime for bot and custom application workloads. Once the request is accepted, VenixCloud begins provisioning immediately and returns the new service object with a `provisioning` status.

<CodeGroup>

```bash cURL
curl -X POST https://api.venixcloud.com/v1/services \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "my-node-bot",
    "type": "bot",
    "region": "us-east-1",
    "runtime": "nodejs-20"
  }'
```

</CodeGroup>

### Request Body

<ParamField body="name" type="string" required>
  A unique, human-readable name for your service. Must be between 3 and 63 characters and contain only lowercase letters, numbers, and hyphens.
</ParamField>

<ParamField body="type" type="string" required>
  The category of service to create. Accepted values are `database`, `bot`, `website`, and `custom`.
</ParamField>

<ParamField body="region" type="string" required>
  The deployment region for this service. Accepted values are `sa-east-1` (São Paulo) and `us-east-1` (North Virginia).
</ParamField>

<ParamField body="runtime" type="string">
  The runtime environment for the service. Examples include `nodejs-20` and `python-3.11`. Required for `bot` and `custom` service types.
</ParamField>

### Response

**Status: `201 Created`**

```json
{
  "id": "svc_xyz789",
  "name": "my-node-bot",
  "type": "bot",
  "status": "provisioning",
  "region": "us-east-1",
  "runtime": "nodejs-20",
  "created_at": "2024-03-10T08:45:00Z"
}
```

<ResponseField name="id" type="string">
  The unique identifier for the newly created service.
</ResponseField>
<ResponseField name="name" type="string">
  The name you provided in the request body.
</ResponseField>
<ResponseField name="type" type="string">
  The service type you specified.
</ResponseField>
<ResponseField name="status" type="string">
  Always `provisioning` immediately after creation. Transitions to `running` once ready.
</ResponseField>
<ResponseField name="region" type="string">
  The region where the service is being deployed.
</ResponseField>
<ResponseField name="runtime" type="string">
  The runtime assigned to the service, if applicable.
</ResponseField>
<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the service was created.
</ResponseField>

## Get a Service

Fetch the full details of a specific service by its ID. This endpoint returns all metadata including the live URL assigned to your service, making it useful for health checks and integration workflows.

<CodeGroup>

```bash cURL
curl -X GET https://api.venixcloud.com/v1/services/svc_abc123 \
  -H "Authorization: Bearer YOUR_API_KEY"
```

</CodeGroup>

### Path Parameters

<ParamField path="id" type="string" required>
  The unique ID of the service you want to retrieve. Service IDs are prefixed with `svc_`.
</ParamField>

### Response

**Status: `200 OK`**

```json
{
  "id": "svc_abc123",
  "name": "my-postgres-db",
  "type": "database",
  "status": "running",
  "region": "sa-east-1",
  "url": "https://svc-abc123.venixcloud.com",
  "created_at": "2024-01-15T10:30:00Z"
}
```

<ResponseField name="id" type="string">
  The unique identifier of the service.
</ResponseField>
<ResponseField name="name" type="string">
  The human-readable name of the service.
</ResponseField>
<ResponseField name="type" type="string">
  The service category: `database`, `bot`, `website`, or `custom`.
</ResponseField>
<ResponseField name="status" type="string">
  The current operational status of the service.
</ResponseField>
<ResponseField name="region" type="string">
  The region the service is deployed in.
</ResponseField>
<ResponseField name="url" type="string">
  The public URL assigned to the service. Available once the service reaches `running` status.
</ResponseField>
<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the service was created.
</ResponseField>

## Delete a Service

Permanently remove a service from your account. This action immediately stops the service, deallocates all associated infrastructure, and cannot be undone.

<Warning>
  Deleting a service is **irreversible**. All associated data — including databases, uploaded files, environment variables, and deployment history — will be permanently destroyed. Make sure you have backed up anything you need before proceeding.
</Warning>

<CodeGroup>

```bash cURL
curl -X DELETE https://api.venixcloud.com/v1/services/svc_abc123 \
  -H "Authorization: Bearer YOUR_API_KEY"
```

</CodeGroup>

### Path Parameters

<ParamField path="id" type="string" required>
  The unique ID of the service to delete.
</ParamField>

### Response

**Status: `204 No Content`**

An empty response body is returned on success. If the service ID does not exist or does not belong to your account, the API returns a `404 Not Found` error.

<Note>
  After receiving a `204`, the service entry will no longer appear in the `GET /services` list. Any DNS records or subdomains pointing to the deleted service will stop resolving within a few minutes.
</Note>
