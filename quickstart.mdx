VenixCloud is a cloud hosting platform that lets you deploy and manage databases, bots, websites, and custom applications from a single dashboard. Whether you're spinning up a PostgreSQL instance, hosting a Discord bot, or launching a production web app, this guide walks you through every step — from creating your account to making your first API call.

<Steps>
  <Step title="Create your account">
    Head to [venixcloud.io](https://venixcloud.io) and click **Sign Up**. Enter your name, email address, and a secure password. After submitting the form, check your inbox for a verification email and click the confirmation link to activate your account.
  </Step>

  <Step title="Access the dashboard">
    Once your email is verified, log in to be taken directly to your **Dashboard**. The left sidebar gives you quick access to your Services, Billing, and Settings. The main panel shows a summary of your active services, resource usage, and recent activity — everything you need at a glance to monitor and manage your infrastructure.
  </Step>

  <Step title="Create your first service">
    In the dashboard, click the **New Service** button in the top-right corner. You'll be prompted to choose a service type:

    - **Database** — PostgreSQL, MySQL, Redis, and more
    - **Bot** — host Discord, Telegram, or custom bots
    - **Website** — static sites or server-rendered apps
    - **Custom App** — any Docker-compatible workload

    Give your service a descriptive name and select the **region** closest to your users. Click **Continue** to proceed to the configuration step.
  </Step>

  <Step title="Configure environment variables">
    Before deploying, add any environment variables your service needs to run correctly. In the **Environment** tab, click **Add Variable**, then enter a key-value pair — for example, `DATABASE_URL` or `BOT_TOKEN`. All variables are encrypted at rest and injected securely into your service's runtime. You can add, edit, or remove variables at any time, even after deployment.
  </Step>

  <Step title="Deploy">
    With your configuration in place, click the **Deploy** button. VenixCloud will build and provision your service — this typically takes between 30 seconds and 2 minutes depending on the service type. Watch the real-time build logs to track progress. Once the status indicator turns green and shows **Running**, copy your **connection string** (for databases) or **public URL** (for websites and bots) from the service overview panel.
  </Step>

  <Step title="Get your API key">
    To manage your services programmatically, navigate to **Settings → API Keys** in the left sidebar. Click **Generate New Key**, give it a descriptive label (e.g., `ci-pipeline` or `local-dev`), and copy it immediately — it won't be shown again. Use this key to authenticate all requests to the VenixCloud REST API.
  </Step>
</Steps>

<Note>
  Free plan accounts are limited to **1 active service**, **256 MB of RAM**, and **1 GB of storage**. Your service will be automatically paused after **30 minutes of inactivity**. Upgrade to a paid plan at any time to remove these limits and keep your services running continuously.
</Note>

## Your first API calls

With your API key in hand, you can interact with VenixCloud programmatically. The examples below demonstrate the most common operations across every resource type. Replace `YOUR_API_KEY` with the key you generated in Step 6.

### Services

Use the `/services` endpoints to list, create, inspect, deploy, and delete your services.

```bash
# List all services
curl -X GET https://api.venixcloud.io/v1/services \
  -H "Authorization: Bearer YOUR_API_KEY"

# Create a new service
curl -X POST https://api.venixcloud.io/v1/services \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"name": "my-app", "type": "custom", "region": "us-east-1"}'

# Get a specific service by ID
curl -X GET https://api.venixcloud.io/v1/services/{id} \
  -H "Authorization: Bearer YOUR_API_KEY"

# Trigger a deployment for a service
curl -X POST https://api.venixcloud.io/v1/services/{id}/deploy \
  -H "Authorization: Bearer YOUR_API_KEY"

# Delete a service
curl -X DELETE https://api.venixcloud.io/v1/services/{id} \
  -H "Authorization: Bearer YOUR_API_KEY"
```

### Databases

Use the `/databases` endpoints to manage your managed database instances.

```bash
# List all databases
curl -X GET https://api.venixcloud.io/v1/databases \
  -H "Authorization: Bearer YOUR_API_KEY"

# Create a new database
curl -X POST https://api.venixcloud.io/v1/databases \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"name": "my-db", "engine": "postgresql", "region": "us-east-1"}'
```

### Bots

Use the `/bots` endpoints to manage your hosted bots.

```bash
# List all bots
curl -X GET https://api.venixcloud.io/v1/bots \
  -H "Authorization: Bearer YOUR_API_KEY"

# Create a new bot
curl -X POST https://api.venixcloud.io/v1/bots \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"name": "my-discord-bot", "region": "us-east-1"}'
```

### Websites

Use the `/websites` endpoints to manage your hosted websites and web apps.

```bash
# List all websites
curl -X GET https://api.venixcloud.io/v1/websites \
  -H "Authorization: Bearer YOUR_API_KEY"

# Create a new website
curl -X POST https://api.venixcloud.io/v1/websites \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"name": "my-site", "region": "us-east-1"}'
```

### Deployments

Use the `/deployments` endpoints to track build and deployment history.

```bash
# List all deployments
curl -X GET https://api.venixcloud.io/v1/deployments \
  -H "Authorization: Bearer YOUR_API_KEY"

# Get details for a specific deployment
curl -X GET https://api.venixcloud.io/v1/deployments/{id} \
  -H "Authorization: Bearer YOUR_API_KEY"
```

A successful response for any list endpoint returns a JSON array of objects. Each object includes the resource ID, name, type, status, region, and relevant endpoint or connection string. You can use these endpoints to build CI/CD pipelines, automate scaling, and manage your entire infrastructure without opening the dashboard.
